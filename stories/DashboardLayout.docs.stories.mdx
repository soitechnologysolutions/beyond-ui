`DashboardLayout` stitches together the persistent `Sidebar` and `DashboardHeader` components to provide a full-page application shell. Import the bundled CSS once in your host app to ensure all semantic tokens resolve:

<Source
  lang="tsx"
  code={`import '@beyondcorp/beyond-ui/dist/styles.css';`}
/>

Use the layout to compose analytics dashboards, admin tools, or internal platforms where a collapsible navigation rail is required.

## Menu model shape

Menu items are plain objects—perfect for fetching from APIs or colocating with route metadata. Provide an array of items, each with an `id`, `label`, optional `icon`, and optional `children` to create nested navigation groups.

<Source
  lang="ts"
  code={`type SidebarMenuItem = {
  id: string;
  label: string;
  icon?: React.ReactNode;
  href?: string;
  badge?: string;
  children?: SidebarMenuItem[];
};`}
/>

## Key props

- `sidebarMenuItems`: hierarchical menu model rendered inside the sidebar.
- `activeSidebarItem` + `onSidebarItemClick`: control the highlighted navigation node.
- `dashboardHeaderProps`: pass search, breadcrumb, and action content to the header.
- `sidebarTitle`, `sidebarHeaderClassName`, and `sidebarMenuFooter`: tailor the navigation rail to match your brand colors and footer content.

> For design token guidance see `docs/theming.md`. The layout and sidebar surfaces immediately adopt updated `primary`, `secondary`, and `accent` hues.
